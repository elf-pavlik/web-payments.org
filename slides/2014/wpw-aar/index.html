Intro

Web Payments Workshop

W3C

  • March 2014 - W3C held first ever Web Payments Workshop in Paris

  • Outcome: W3C should do something to address payment problems on the Web

  • There is no "one way" to do payments.

  • Focus on small, modular pieces.

Input

Papers

Workshop Paper Survey Findings

Papers

  • Survey of Workshop Papers

  • No Verified Identity / Trust Mechanism (72%)

  • Outdated Security Tech (50%)

  • No Payment Initiation Standard (38%)

  • No Digital Receipt Standard (38%)

Web Payments

CG

Web Payments Community Group


  • W3C Web Payments CG - 160+ members from payments industry, NGOs, and public

Problem

Identity

Workshop Findings on Identity

Chatter

  • Identity - complex field, very basic interoperability

  • Persona, OpenID Connect, not robust enough

  • Need 3rd party, digitally signed credentials

  • Proposal: Identity Credentials

  • Bonus: Works like Persona, so can be used for Web login problem as well

Problem

Initiating

Initiating Payment

  • How do we initiate a payment online?

  • ... and interoperate with FirefoxOS, Paypal, Visa/Mastercard?

  • Need payment initiation standard - like the good parts of the MozPay API

  • Proposal: Web Commerce API

Problem

Receipts

Offers and Proof of Purchase

  • How do we prove that we've purchased something?

  • ... and interoperate with FirefoxOS, Paypal, Visa/Mastercard?

  • Need digital receipt standard - should be very extensible, based on JSON-LD?

  • Proposal: Web Commerce
