The Problem

  • We have a universal identifier for phone, e-mail and websites.
  • We don't have one for identifying people and their preferences on the Web.
  • Username/passwords becoming insecure - rainbow table attacks.
  • We are not in control of our own identity on the Web.

Potential solutions...

Potential Solutions

  • Universal identification on the Web - OpenID Connect, WebFinger, and WebID.
  • All seem to be converging on a username/password-less way to log into websites.
  • Many are concerned with transmitting name, address and friend information.

WebID demo...

WebID

  • Purpose: Web architecture-compliant identity and authentication protocol for the Social Web.
  • Description of how WebID works vs. how OpenID works
  • Advantages
    • Usability: No password or URL to remember, no need to repeatedly enter friends and contacts, potential for browser-integration, very high security (TLS)
    • Technical: wide support in desktop browsers, RESTful authentication, peer-to-peer identity, simplified identification protocol, supports digital signatures (X509), supports strong encryption of messages (TLS), extensibility via Linked Data (RDF, RDFa), syntax agnostic (easy to integrate w/ Portable Contacts, etc.)
  • Disadvantages: Inability to log out, user-interfaces, uniform UI across all browsers, does not work on older cellphones - patchy support on newer ones

The W3C work...

Web Identity Browser API

Implementation issues that will lead to the Web Identity Browser API...

  • Cross-domain communication with WebID end point (Flash, Flash cross domain policies, Apache module)
  • Communication between Javascript and Flash (base64 encode strings to pass utf-16 data)
  • Lack of a raw socket interface (Flash raw sockets)
  • Private Key/Certificate local storage (Flash, HTML5 Local Storage)
  • Cross-domain login communication (iframe, Flash, iframe data passing hacks)
  • TLS implementation in Javascript (pure Javascript/Flash)
  • Certificate/key export (Flash to save/load binary data)

Potential Impact

Learn More about Identity on the Web

  • WebID (Wiki) - Decentralized Identity and Discovery for the Web.
  • OpenID - Open identity for the Web.
  • OpenID Connect - The newest development work on OpenID - effectively OpenID 3.0.
  • OAuth - Authorization of Web Agents to operate on one's behalf.
  • XAuth - Cross-site preferences cookie jar - solves NASCAR problem.
  • WebFinger - Discovery of identity information via e-mail addresses
  • /.well-known/ URIs - Directory for well-known files such as robots.txt, icon files, avatar images, etc.
  • Portable Contacts - Universal friends and contacts, remove friend entry repetition.
  • Activity Streams - A format for syndicating social activities around the Web.
  • Open Social - Open APIs for social network interoperability.
