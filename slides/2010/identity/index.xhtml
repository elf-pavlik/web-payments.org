Implementation Issues

  • Cross-domain communicating with the end point (flash, cross domain policies, apache module to solve this, etc).
  • Communication between javascript and flash (needing to base64 encode strings to pass utf-16 strings between them).
  • Needing a raw socket ... needed to use flash to do this.
  • Local storage ... needed for pre-IE8 browsers.
  • Cross-browser storage - one cert works in all browsers.
  • Cross-domain communicating/interfacing with web ID provider on the end service's website (iframe data passing).
  • Needing to implement TLS in javascript due to lack of crypto/TLS-cert APIs in the browser. (More like a reason why it had to happen vs. implementation difficulty?)
  • Needing flash to save/load binary files (we don't even do this yet, but it should work).

Learn More about Identity on the Web

  • WebID (Wiki)
    • Decentralized Identity and Discovery for the Web.
  • OpenID
    • Open identity for the Web.
  • OpenID Connect
    • The newest development work on OpenID - effectively OpenID 3.0.
  • OAuth
    • Authorization of Web Agents to operate on one's behalf.
  • XAuth
    • Cross-site preferences cookie jar - solves NASCAR problem.
  • WebFinger
    • Discovery of identity information via e-mail addresses
  • /.well-known/ URIs
    • Directory for well-known files such as robots.txt, icon files, avatar images, etc.
  • Portable Contacts
    • Universal friends and contacts, remove friend entry repetition.
  • Activity Streams
    • A format for syndicating social activities around the Web.
  • Open Social
    • Open APIs for social network interoperability.
